Essensielle sjekklister, progresjonsrammeverk og dokumentasjonslenker for løpande implementering og meistring av AI Native Development.

## Hurtigstartsjekkliste {#quick-start-checklist}

### Konseptuelt grunnlag
1. **[ ]** Forstå prinsippa for **Markdown Prompt Engineering** (semantisk struktur + presisjon + verktøy)
2. **[ ]** Forstå grunnleggjande **Context Engineering** (bufferoptimalisering + øktstrategi)
3. **[ ]** Lær **Skills som distribusjon** (kapabilitetar pakka for automatisk oppdaging)

### Implementeringssteg
4. **[ ]** Installer nyttige Skills for din stack: `apm install owner/skill-name` (Skills: umiddelbare kapabilitetar)
5. **[ ]** Opprett [`.github/copilot-instructions.md`](https://code.visualstudio.com/docs/copilot/copilot-customization#_use-a-githubcopilot-instructionsmd-file) med grunnleggjande prosjektretningslinjer (Context Engineering: globale reglar)
6. **[ ]** Sett opp domenespesifikke [`.instructions.md`-filer](https://code.visualstudio.com/docs/copilot/copilot-customization#_use-instructionsmd-files) med `applyTo`-mønster (Context Engineering: selektiv lasting)
7. **[ ]** Konfigurer [tilpassa agentar](https://code.visualstudio.com/docs/copilot/copilot-customization#_custom-chat-modes) for domena i teknologistakken (Context Engineering: domenebegrensingar)
8. **[ ]** Lag den første [`.prompt.md`-fila](https://code.visualstudio.com/docs/copilot/copilot-customization#_prompt-files-experimental) med valideringssjekkpunkter (Markdown Prompt Engineering: deterministiske malar)
9. **[ ]** Bygg den første `.spec.md`-malen for funksjonsspesifikasjonar (Agent Primitive: deterministisk bru frå plan til implementering)
10. **[ ]** Pakk gjenbrukbare mønster som ein Skill: `apm init skill` (Skills: deling av kunnskap)
11. **[ ]** Praktiser spec-first-arbeidsflyt: planlegg først, implementer deretter (Context Engineering: deling av økter)
12. **[ ]** Test asynkron delegering med [GitHub Coding Agent](https://docs.github.com/en/copilot/about-github-copilot/about-copilot-coding-agent) (avansert orkestrering)
13. **[ ]** Etabler teamstyring og valideringsportar (menneske–AI-samarbeidsmønster)

## Meistringsprogresjon {#mastery-progression}

### Grunnnivå
**Mål:** Forstå kjerneomgrep og bygg dei første primitivane  
**Tidsbruk:** 2–3 timar  
**Nøkkelutfall:**
- [ ] Oppretta grunnleggjande `.instructions.md`-filer
- [ ] Konfigurert første chat-modus
- [ ] Bygd første `.prompt.md`-mal
- [ ] Forstår det teoretiske rammeverket

**Neste steg:** Gå til nybyrjarnivå med praktisk implementering

### Nybyrjarnivå  
**Mål:** Grunnleggjande instruksjonar og prompts fungerer konsistent  
**Tidsbruk:** 4–6 timar  
**Nøkkelutfall:**
- [ ] Fungerande domenespesifikke instruksjonar
- [ ] Fleire chat-modi konfigurert
- [ ] Promptbibliotek med 3–5 malar
- [ ] Konsistente AI-interaksjonar i dagleg arbeid

**Neste steg:** Avanser til mellomnivå med arbeidsflytmønster

### Mellomnivå
**Mål:** Spec-drivne arbeidsflytar med kontekstoptimalisering  
**Tidsbruk:** 8–12 timar  
**Nøkkelutfall:**
- [ ] Spec-first planleggingsmetodikk
- [ ] Kontekstingeniørstrategiar implementert
- [ ] Deling av økter for komplekse oppgåver
- [ ] Minnedrivne utviklingsmønster

**Neste steg:** Progresjon til avansert nivå med asynkron delegering

### Avansert nivå
**Mål:** Asynkron delegering og multiagent-orkestrering  
**Tidsbruk:** 15–20 timar  
**Nøkkelutfall:**
- [ ] GitHub Coding Agent-delegering fungerer
- [ ] Parallelle multiagent-arbeidsflytar
- [ ] Kvalitetsportar og valideringsprosessar
- [ ] Hybride kontekststrategiar

**Neste steg:** Oppnå ekspertnivå med teamimplementering

### Ekspertnivå
**Mål:** Teamomfattande styring og innovasjon i frontier-mønster  
**Tidsbruk:** 25+ timar  
**Nøkkelutfall:**
- [ ] Koordineringsrammeverk i teamskala
- [ ] Kunnskapsdelingssystem
- [ ] Styring og compliance-integrasjon
- [ ] Innovasjon i AI Native Development-mønster

**Neste steg:** Bidra til fellesskapskunnskap og frontier-forsking

## Paradigmeskiftet {#the-paradigm-shift}

*Tradisjonell tilnærming: "Fortel AI kva den skal gjere"*  
**Agent Mastery-tilnærming: "Konstruer kontekst og struktur for optimal kognitiv ytelse"**

### Kjerneprinsipp
1. **Determinisme gjennom struktur:** Meir føreseielege resultat gjennom systematiske tilnærmingar
2. **Kontekst som ytelse:** Strategisk minnehandtering for optimal kognitiv ytelse i AI
3. **Komponert intelligens:** System som forbetrar seg gjennom iterasjon og læring
4. **Menneske–AI-partnerskap:** Valideringsportar og samarbeidsarbeidsflytar
5. **Koordinering i teamskala:** Kunnskapsdeling og organisatorisk transformasjon

### Nøkkelinnsikter
- **Jo meir determinisme du treng, desto meir Markdown Prompt Engineering og mindre omfang må du bruke**
- **Jo meir komplekst prosjektet ditt er, desto meir kritisk blir Context Engineering**
- **Meistring av begge prinsippa gir eineståande konsistens og kvalitet i agentdriven utvikling**

**Hugs:** Start enkelt, iterer raskt, skaler systematisk gjennom systematisk bruk av desse frontier-konsepta.

## Hurtig feilsøking {#quick-troubleshooting}

### Vanlege problem og løysingar

**Problem:** AI-svar er inkonsistente  
**Løysing:** Implementer meir strukturerte Markdown-prompts med tydelege overskrifter og valideringsportar

**Problem:** Avgrensingar i kontekstvindauget  
**Løysing:** Bruk øktdeling og modulære instruksjonar med `applyTo`-mønster

**Problem:** Teamkoordinering gir konfliktar  
**Løysing:** Etabler delte primitivbibliotek og koordineringsprotokollar for repo

**Problem:** Kvalitetsbekymringar med asynkrone agentar  
**Løysing:** Implementer valideringsportar og behandl agentutdata som høgkvalitetsutkast som må gjennomgåast

**Problem:** Compliance- og tryggleiksbekymringar  
**Løysing:** Bruk risikobaserte agentgrenser og bygg policykrav inn i instruksjonar
